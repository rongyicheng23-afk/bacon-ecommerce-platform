# Bacon Mall 商品数据维护指南

这份文档用于给项目组成员维护商城中的商品。项目目前不使用爬虫，商品名称、价格、分类、库存、描述和图片均由我们手动准备。

## 先看结论

- 只改已有商品的价格、库存：使用网页的商家中心，最简单。
- 要批量修改商品名称、描述、分类、图片，或替换整批商品：修改 `backend/app/db/seed.py`。
- 不要直接修改前端页面里的商品卡片。前端只负责展示，商品数据来自 FastAPI + SQLite。

## 方法一：在网页商家中心修改价格和库存

适用于日常演示时的小调整，不会清空订单、行为日志或推荐结果。

1. 启动前端和后端。

   ```bash
   cd /Users/rongyicheng/Desktop/专业实习2-项目/bacon-mall-frontend
   npm run dev -- --host 127.0.0.1

   cd backend
   source .venv/bin/activate
   uvicorn app.main:app --reload --host 127.0.0.1 --port 8001
   ```

2. 浏览器打开 `http://127.0.0.1:5173/`。
3. 使用商家账号登录：

   ```text
   seller@example.com
   123456
   ```

4. 进入“商家中心” -> “商品管理”。
5. 找到目标商品，点击“编辑”，修改价格和库存后保存。

说明：当前商家中心已经接入真实后端，因此修改会写入 SQLite 数据库，刷新页面后仍然有效。

## 方法二：批量手动维护商品内容

适用于准备答辩商品库，例如把重复商品改成不同的数码、服饰、家居、食品、美妆或图书商品。

### 1. 打开商品种子文件

在 VS Code 中打开：

```text
backend/app/db/seed.py
```

文件顶部的 `PRODUCT_SEEDS` 是商品清单。每一条商品的格式是：

```python
("商品名称", "商品描述", 价格, "分类", ["图片地址1", "图片地址2", "图片地址3", "图片地址4"]),
```

示例：

```python
("蓝牙降噪耳机", "适合通勤和学习，支持主动降噪与通透模式。", 199, "数码", [
    "https://images.unsplash.com/photo-1505740420928-5e560c06d30e?auto=format&fit=crop&w=900&q=85",
    "https://images.unsplash.com/photo-1583394838336-acd977736f90?auto=format&fit=crop&w=900&q=85",
]),
```

### 2. 分类只能使用这些名称

为了让分类页、推荐系统和商品筛选正常工作，分类请使用以下任意一个：

```text
数码、服饰、家居、运动、食品、美妆、图书
```

不要写“数码家电”“化妆品”“零食”等新的分类名称，除非同时修改分类表和前端分类配置。

### 3. 图片怎么填

演示阶段可以使用自己上传的商品图片，或有公开使用许可的图片链接。每个商品建议至少填 2 张图片，第一张会作为商品列表主图。

图片与商品必须对应。例如：

- 耳机使用耳机图。
- 背包使用背包图。
- 护肤品使用护肤品图。

不要把同一张耳机图重复用于面霜、背包、台灯等无关商品，否则商品页会显得不真实。

## 使用 MinIO 保存商品图片（推荐）

商品图片不应该直接保存在 SQLite 里，也不需要保存到 Hadoop。正确分工如下：

| 内容 | 存储位置 |
| --- | --- |
| 商品名称、价格、库存、分类、描述 | SQLite 的商品表 |
| 图片文件、视频文件 | MinIO 对象存储 |
| 商品图片 URL | SQLite 的 `products.image_urls` 字段 |
| 浏览、收藏、加购、购买日志 | SQLite，后续导出到 HDFS |

### 1. 启动 MinIO

打开一个新的终端：

```bash
cd /Users/rongyicheng/Desktop/专业实习2-项目/bacon-mall-frontend/backend
bash scripts/start_minio.sh
```

启动后：

```text
MinIO 控制台：http://127.0.0.1:9001
MinIO API：http://127.0.0.1:9000
账号：minioadmin
密码：minioadmin
```

如果终端提示找不到 `minio` 命令，先安装：

```bash
brew install minio/stable/minio
```

### 2. 配置后端使用 MinIO

打开 `backend/.env`，确认包含：

```env
MINIO_ENDPOINT=127.0.0.1:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_SECURE=false
```

修改后重启 FastAPI 后端。后端会自动创建 `product-images`、`avatars`、`shop-logos` 三个 bucket。

### 3. 上传图片并取得 URL

推荐做法是通过后端接口上传，因为系统会记录上传者，商家只能删除自己的图片。

1. 使用商家账号登录前端。
2. 打开 `http://127.0.0.1:8001/docs`。
3. 找到 `POST /api/media/upload`，点击 `Try it out`。
4. `folder` 填 `products`，选择本地图片文件。
5. 在 `authorization` 中填写：

   ```text
   Bearer 你的登录 token
   ```

6. 执行后复制响应里的 `url`。

登录 token 可在浏览器开发者工具的 Application（或“应用”）-> Local Storage -> `token` 中找到。

上传完成后，响应中的 URL 形式类似：

```text
http://127.0.0.1:9000/product-images/products/xxxxxxxxxxxxxxxx.jpg
```

### 4. 把图片 URL 填入商品

将上一步得到的 URL 填到 `backend/app/db/seed.py` 中商品的图片数组里：

```python
("蓝牙降噪耳机", "适合通勤和学习。", 199, "数码", [
    "http://127.0.0.1:9000/product-images/products/耳机主图.jpg",
    "http://127.0.0.1:9000/product-images/products/耳机详情图.jpg",
]),
```

之后按“让批量修改生效”部分重建开发数据库。图片文件仍保留在 MinIO，数据库只保存图片 URL。

## 让批量修改生效

`seed.py` 只会补充数据库中不存在的商品，不会自动覆盖已经存在的商品。因此，修改了商品种子数据后，在开发演示环境需要重建本地数据库。

### 重建前先备份

下面操作会清空本机的测试账号、购物车、订单、行为日志和推荐结果。不要对答辩前需要保留的数据直接执行。

```bash
cd /Users/rongyicheng/Desktop/专业实习2-项目/bacon-mall-frontend
cp backend/bacon_mall.db backend/bacon_mall.backup.db
```

### 重建数据库

1. 先停止后端服务。
2. 删除旧的开发数据库：

   ```bash
   rm backend/bacon_mall.db
   ```

3. 重新启动后端：

   ```bash
   cd backend
   source .venv/bin/activate
   uvicorn app.main:app --reload --host 127.0.0.1 --port 8001
   ```

4. 后端启动时会自动创建数据库、分类、测试账号、店铺、商品和默认收货地址。
5. 刷新前端页面，确认商品列表已更新。

## 推荐的人工商品数据规模

为了让商城看起来像一个正常电商平台，建议准备：

| 内容 | 建议数量 |
| --- | ---: |
| 商品总数 | 80 到 150 个 |
| 每个大分类 | 10 个以上 |
| 每个商品图片 | 2 到 4 张 |
| 每个商品 SKU | 系统会自动生成颜色和版本 SKU |

Hadoop 推荐系统的重点是用户行为日志，不是商品必须来自真实网站。商品数据手动维护完全符合本项目目标。

## 修改后检查清单

1. 商品名称、描述、图片是否匹配。
2. 商品分类是否属于允许的 7 个分类。
3. 价格是否为正数，库存是否合理。
4. 商品列表是否有大量重复图片或重复名称。
5. 商品详情页能否正常打开。
6. 商品能否加入购物车并完成下单。
7. 重新生成推荐后，首页是否能看到相应分类商品。

## 常见问题

### 改了 `seed.py`，网页商品没有变化

这是正常的，因为旧 SQLite 数据库仍存在。按照“让批量修改生效”部分备份并重建数据库。

### 页面图片加载失败

先检查图片链接能否在浏览器中单独打开。建议使用 HTTPS 图片地址，避免使用需要登录、会过期或禁止外链的图片。

### 不想重建数据库，只想改一两个商品

直接使用商家中心修改价格和库存；商品名称、描述、分类和图片的编辑功能可以在后续继续扩展。
